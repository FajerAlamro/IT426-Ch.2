01/06

Observability

Full visibility vs hidden information

Fully Observable

Complete state visibility

The agent can see the entire state of the environment at every moment. Nothing is hidden — all pieces, positions, and moves are visible to both players.
♟️ Watch both sides play — every piece and move is visible to everyone
White is thinking...
✅
When is it used?
When the agent needs complete knowledge to make optimal decisions — board games, security surveillance, factory monitoring, or any system where all data is accessible in real time.
Chess BoardSecurity CamerasPuzzle Game

Partially Observable

Hidden information

The agent can only see part of the environment. Key information like prices and ratings are hidden until you interact — decisions are made with incomplete data.
👆 Click products to reveal details
✅
When is it used?
When the agent cannot access full state — medical diagnosis (hidden symptoms), poker (hidden cards), or real-world robotics where sensors have limits. Agent must reason under uncertainty.
Online ShoppingMedical DiagnosisStock Trading
02/06

Determinism

Predictable vs random outcomes

Deterministic

Same action, same result

The outcome is 100% predictable. Withdraw $200 and you'll always get exactly $200 — no surprises, no randomness.
👆 Enter PIN and withdraw
✅
When is it used?
When outcomes are fully predictable and repeatable — calculators, vending machines, alarm clocks, or any rule-based system where the same input always produces the exact same output.
ATM WithdrawalCalculatorAlarm Clock

Stochastic

Random outcomes

The same conditions can produce different outcomes each time. Refresh the board — flight statuses change randomly due to unpredictable factors like weather.
🔄 Refresh to see status changes
✅
When is it used?
When real-world uncertainty is involved — weather prediction, traffic flow, customer behavior, or any environment affected by random external factors the agent cannot fully control.
Flight DelaysNetwork SpeedCustomer Arrival
03/06

Episodic vs Sequential

Independent vs connected actions

Episodic

Independent tasks

Each decision is independent. Classifying one email as spam has zero effect on the next one — the order doesn't matter at all.
🔄 Watch — each email classified independently (random order)
✅
When is it used?
When each task is self-contained and isolated — spam filters, image classifiers, product quality inspection, or password validation. No memory of past decisions needed; each episode is complete on its own.
Spam DetectionProduct ReviewPassword Check

Sequential

Connected decisions

Each step builds on the result of the previous one. You can't choose investments before assessing risk — the order is critical and decisions are chained.
🔗 Watch — each step depends on the previous result
✅
When is it used?
When decisions affect future options — chess, robotic navigation, investment planning, or medical treatment steps. Each action has long-term consequences that must be considered before acting.
Investment PlanningCareer ProgressionHouse Renovation
04/06

Static vs Dynamic

Waiting vs changing environment

Static

Environment waits

The environment doesn't change while you're deciding. Take as long as you want — nothing moves until you make your move.
👆 Play at your own pace — nothing changes until you move
✅
When is it used?
When the environment is frozen until the agent acts — crossword puzzles, photo editing, Sudoku, or theorem proving. The agent can deliberate without time pressure or fear of the world shifting mid-decision.
Crossword PuzzlePhoto EditingSudoku

Dynamic

Constant change

The environment keeps changing whether you act or not. Stock prices move on their own — if you wait too long, the opportunity might be gone.
📈 Watch prices change in real-time — you can't pause the market!
✅
When is it used?
When the world keeps changing independently — self-driving cars, live trading bots, online multiplayer games, or real-time robotics. The agent must act quickly because hesitation itself changes the environment.
Stock MarketSelf-Driving CarOnline Multiplayer Game
05/06

Discrete vs Continuous

Limited vs infinite choices

Discrete

Countable options

You choose from a finite set of options — meaning the choices are fixed, countable, and cannot be divided. There are exactly 9 floors: 1, 2, 3... up to 9. You can't press 3.5 or go to floor 6.7
👆 Press floor buttons — watch the elevator move!
✅
When is it used?
When states or actions are countable: menu buttons, number of people, or chess moves — each is a distinct, countable option with no in-between values.
Elevator ButtonsMenu SelectionVote Choices

Continuous

Infinite precision

Actions exist on a continuous range with infinite possible values. The drone can move to any point in space — coordinates like x=0.347, y=-0.891 are all valid.
👆 Click anywhere — infinite possible positions
✅
When is it used?
When values are continuous and cannot be enumerated — weight, speed, rotation angle, temperature, or spatial position. Used in robotics, physics simulations, and any environment requiring fine-grained precision.
Robot Arm MovementDrone NavigationSelf-Driving Steering
06/06

Single vs Multi-Agent

Alone vs with others

Single-Agent

Working alone

Only one agent operates in the environment. You scan and pay by yourself — no other agent affects your experience or decisions.
👆 Scan items yourself
✅
When is it used?
When only one decision-maker exists — ATM transactions, solo puzzle solving, self-checkout, or a personal AI assistant that doesn't need to model or compete with other agents.
Online LearningATM TransactionSelf Checkout

Multi-Agent

Multiple participants

Multiple agents share the same environment. Each participant's actions affect others — coordination, competition, and negotiation come into play.
👥 View team members
✅
When is it used?
When multiple agents interact and influence each other — autonomous vehicles on roads, business negotiations, multiplayer games, or AI bots on a shared platform. Agents may cooperate or compete.
Business NegotiationTraffic NavigationTeam Meeting

Quick Review: Environment Properties
1
Observability
Fully Observable
Agent sees the complete state of the environment. Example: Chess — all pieces visible.
Partially Observable
Some information is hidden from the agent. Example: Poker — you can't see opponents' cards.
2
Determinism
Deterministic
Same action always produces the same result. Example: ATM — withdraw $100, get exactly $100.
Stochastic
Outcomes involve randomness or uncertainty. Example: Flights — same route, different delays each day.
3
Episodic vs Sequential
Episodic
Each action is independent — previous decisions don't affect the next. Example: Spam filter classifying emails.
Sequential
Current decisions depend on and affect future ones. Example: Investment steps — each builds on the last.
4
Static vs Dynamic
Static
Environment doesn't change while the agent is deciding. Example: Tic-Tac-Toe — board waits for your move.
Dynamic
Environment changes on its own while agent decides. Example: Stock market — prices keep moving.
5
Discrete vs Continuous
Discrete
Finite number of actions or states. Example: Elevator — only specific floor buttons 1, 2, 3…
Continuous
Infinite possible values in actions or states. Example: Drone — can move to any coordinate in space.
6
Single vs Multi-Agent
Single-Agent
Only one agent operates in the environment. Example: Self-checkout — you work alone.
Multi-Agent
Multiple agents interact and affect each other. Example: Business negotiation — competing goals.
